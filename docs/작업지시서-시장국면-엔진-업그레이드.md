## 3. Phase 1 — 시장국면 판별 엔진 (요구사항 1·3)

### 3-1. 공유 모듈 `src/lib/marketRegime.ts`

- 대상: KOSPI(^KS11)·KOSDAQ(^KQ11) 각각. 일봉 1y(MA120 + ADX 워밍업 확보).
- 선행: ADX(14)·ATR(14) 헬퍼 추가(§5-1 지표 확장과 공용).
- 판정 (원문 조건 + 고도화):

| 국면 | 조건 (충족 개수) | 고도화 포인트 |
|---|---|---|
| 상승 BULL | 6중 4↑: Close>MA20 · MA20>MA60 · MA60>MA120 · ADX>25 · 20일 신고가 근접(고가 대비 -2% 이내) · 거래량>20일 평균 1.2배 | "20일 신고가"를 당일 정확 갱신이 아닌 근접(-2%)으로 완화해 노이즈 감소 |
| 횡보 SIDEWAYS | 5중 3↑: MA20 기울기 ±0.15%/일 · ADX<20 · 20일 변동폭 ≤15% · ATR(14) 20일 평균 대비 감소 · 거래량 20일 평균 이하 | 기울기 임계 수치화 |
| 하락 BEAR | 6중 4↑: Close<MA20 · MA20<MA60 · MA60<MA120 · ADX>25 · 20일 신저가 근접(+2%) · 거래량>평균 1.2배 | |

- **우선순위**: 복수 충족 시 BEAR > BULL > SIDEWAYS (방어 우선). 전부 미충족 → SIDEWAYS 폴백.
- **히스테리시스(고도화)**: 직전 close 세션 국면과 다른 국면으로 바뀌려면 새 국면 충족 개수가 기준 +1 이상이어야 전환(경계에서 하루 단위 국면 널뛰기 방지). preopen/midday는 히스테리시스 없이 참고치로 기록.
- 반환: `{ market, regime, score:{bull,sideways,bear}, evidence:[{label,met,value}] }` — UI 근거·텔레그램 본문 공용.
- 대표 국면 = KOSPI. 두 시장 상이하면 UI에 병기.

### 3-2. 스케줄 함수 `netlify/functions/regime.mts`

- `schedule: '*/10 * * * *'` + 내부에서 KST 08:30/11:00/15:30 ±5분 창 판정. 해당 세션 기록이 이미 있으면 skip(idempotent). 주말 스킵.
- 처리: 판별 → `bnf_market_regime` insert → 수신 설정 켠 사용자에게 텔레그램 발송.

### 3-3. DB (schema.sql)

```sql
create table if not exists public.bnf_market_regime (
  id bigserial primary key,
  judged_at timestamptz not null default now(),
  session text not null check (session in ('preopen','midday','close')),
  trade_date date not null,
  kospi_regime text not null,
  kosdaq_regime text not null,
  detail jsonb not null default '{}',
  unique (trade_date, session)
);
-- RLS: 로그인 사용자 select / service role insert
```

### 3-4. 텔레그램 설정 (SettingsPage)

- `bnf_profiles.settings.telegram.regimeNotify = { enabled, preopen, midday, close }` (기본 false).
- 체크박스 4개(수신 여부 + 세션 3개). 발송 본문: 국면 이모지 + KOSPI/KOSDAQ + 충족 근거 요약 + 추천 전략 1줄.
- AC: 세션 체크 끄면 미발송 · 주말 미발송 · 동일 세션 중복 발송 없음.

## 5. Phase 3 — 공통 고도화 프레임 + 전략 9종

### 5-1. 공통 프레임 (모든 전략에 일괄 적용 — 선행 커밋)

**(a) 지표 확장** — `indicators.ts`(또는 engine.ts)에 추가: `ATR(n)`, `ADX(n)`, `CCI(n)`, `SlowStochastic(k,d)`, `OBV`, `당일 VWAP(15m 누적)`, `volumeZScore(20)`, 캔들패턴 판별 `detectCandle(row, prev)` → 망치형/장악형/샛별형/장대양봉/긴 아래꼬리(몸통 2배↑ 꼬리) — OHLC 비율 기반.

**(b) StrategyModule 메타 확장** — `types.ts`:
```ts
regime: 'BULL' | 'SIDEWAYS' | 'BEAR' | 'ANY';
risk: 1 | 2 | 3 | 4 | 5;
maxHoldBars?: number;        // 시간청산 (봉 수)
minScore?: number;           // 자동매매 진입 최소 스캔점수 (기본 70)
```

**(c) 국면 게이트** — trader.mts 매수 루프: 진입 전 최신 `bnf_market_regime`(없으면 실시간 계산) 확인, `mod.regime` 불일치 시 해당 전략 매수 스킵(+로그). **매도/손절 루프는 국면 무관 항상 실행**(기존 원칙 유지). 스캐너/대시보드 scan에도 "시장 국면 일치" 조건 행을 추가해 표기.

**(d) 공통 리스크 필터** (planEntry 헬퍼로 공용화):
- 손익비 필터: (1차 목표가-진입가)/(진입가-손절가) ≥ 1.5 미만이면 스킵 (bnf1의 minExpectedReturn 개념 일반화)
- 변동성 상한: 당일 ATR(14) > 20일 평균 ATR × 2.5 → 스킵
- 갭 필터: 당일 시가 갭 < -5% 또는 > +5% → 스킵
- 과열 제외: 최근 5일 수익률 ≥ +25% 또는 RSI(14) ≥ 80 → 스킵
- 교차 전략 중복 방지: 동일 종목을 타 전략이 OPEN 보유 중이면 스킵 (trader.mts heldSymbols를 전략 구분 없이 전 계정 범위로 확대)

**(e) 청산 공통화**: ATR 트레일링 헬퍼 `atrTrail(pos, row, mult)` + 시간청산(`maxHoldBars` 경과 시 전량) stepOpen 헬퍼 제공.

### 5-3. 전략별 확정 스펙 (md보다 고도화된 버전 — 이대로 구현)

> 표기: 진입은 "필수 AND 조건" + "점수 조건(scan score 가중치는 구현 재량, 총 100점)". 공통 리스크 필터(§5-1d)는 전 전략 자동 적용이므로 생략.

**S1. breakout — (상승) 추세추종 · 신고가 돌파** `1d · BULL · risk3 · maxHold 없음`
- 진입 필수: 정배열(MA20>MA60>MA120) · Close>MA20 · **60일 신고가 종가 돌파**(또는 20일 신고가+ADX≥30) · 거래량 Z≥2.0 · 돌파봉 윗꼬리 ≤ 캔들범위 30% · RS(20일 수익률 > KOSPI 20일 수익률) · OBV 20일 기울기>0
- 손절: max(-6%, 돌파봉 저가, 진입가-ATR14×2)
- 청산 사다리: 수익 +8% 도달 후 → 10일선 종가 이탈 30% → 20일선 이탈 30% → 잔여 ATR×3 트레일 또는 60일선 이탈 전량
- 고도화 요지: 단순 저항 돌파 → 신고가+RS+OBV 3중 확인, 고정 EMA20 청산 → 3단 트레일링

**S2. pullback — (상승) 눌림목 스윙** `1d · BULL · risk2 · maxHold 7`
- 진입 필수: 정배열(MA5>MA20>MA60) · 20일 고점 대비 -3%~-8% 조정 · 종가가 MA10~MA20 밴드 내 · 조정기 5일 평균 거래량 ≤ 20일 평균 70% · 반등일 거래량 150%↑ · 반등캔들(장대양봉/망치/장악형) & 종가>전일 고가 · RSI(14) 45~65 · MACD 히스토그램 증가
- 손절: max(-4%, MA20 종가 이탈, ATR×1.5)
- 익절: +5% 30% → +8% 30% → 잔여 MA5 종가 이탈 전량
- 시간청산: 7거래일 경과 & 수익 3% 미만 → 전량
- 고도화 요지: 스토캐스틱 단일 트리거 → "건강한 조정(거래량 감소)" + 반등 확인형 진입

**S3. alignment — (상승) 이동평균 정배열** `1d · BULL · risk2`
- 기존 로직 유지 + 강화: 정배열 **완성 후 3봉 이내만** 진입(초입 한정) · ADX≥20 추가 · 진입일 거래량 ≥ 20일 평균 · 이격도(Close/MA20) ≤ 1.08 유지
- 청산: 기존(20일선 이탈 50%, 60일선/데드크로스 전량) 유지하되 Phase 0 라벨 규칙 적용

**S4. openbrk — (상승) 시가돌파 단타 [신규]** `15m · BULL · risk5 · 당일 청산`
- 전제 명시: 15분봉 근사 구현(rules에 "실시간 초단위 아님" 표기). Yahoo 지연 감안, KIS 실시간 연동 사용자에게 유리함을 설명.
- 대상 선별: 전일 양봉 & 전일 거래량 Z≥1.5 종목
- 진입(09:00~10:30 봉만): 15분봉 종가가 당일 시가 +1% 상향 돌파 · 당일 누적 VWAP 위 · 돌파봉 거래량 Z≥2 · 시가 갭 -1%~+3% 범위
- 손절: 시가 -1.5% 또는 진입봉 저가 중 높은 값
- 익절: +2% 40% → +4% 30% → 잔여 VWAP 종가 이탈 전량
- **강제 청산: 15:20 이후 잔여 전량 시장가** — trader.mts 매도 루프에 시각 게이트 추가
- 추천 로직에서 자동 제외(risk 5)

**S5. bnf1 — (횡보) 볼린저 수렴 회귀** `15m · SIDEWAYS · risk2`
- 로직 변경 없음(최근 고도화 완료: 최소 기대수익 0.8% 필터, TP1 후 중심선 이탈 마감 청산). 메타(regime/risk)와 국면 게이트만 추가. BEAR 국면에서 신규 진입 금지 효과.

**S6. rangeswing — (횡보) 박스권 스윙 [신규·2md 통합]** `1d · SIDEWAYS · risk3 · maxHold 5`
- 박스 정의: 최근 25일 고저. 폭 6~18% · 하단 터치(±1.5%) 2회↑ · 상단 터치 2회↑ · 수렴형(고점 하락 & 저점 상승 동시 진행) 제외
- 박스 신뢰도 점수(스캔 점수의 골격): 터치 횟수·유지 기간·폭 적정성 가중
- 진입 필수: 종가가 박스 하단 +2% 이내 & 하단 종가 미이탈 · 조정기 거래량 감소(5일≤20일 70%) · 반등캔들 · RSI(14) 30~50 · Slow Stoch %K<30 골든크로스 · CCI -100 상향 회복
- 손절: 박스 하단 -2% 종가 이탈 또는 -3%
- 익절: 박스 중앙 40% → 상단 -1% 40% → 잔여: 상단 돌파 시 ATR×1.5 트레일 / 3봉 내 돌파 실패 시 전량
- 시간청산: 5거래일 & 수익 2% 미만 전량
- 고도화 요지: 단타 md(20일 박스)와 스윙 md(30일 박스)를 25일로 통합, 잔여 물량의 "돌파 전환 트레일"로 박스 돌파 수익까지 흡수

**S7. box — (횡보) 박스권 돌파** `1d · SIDEWAYS · risk3`
- 기존 로직 유지 + 강화: 돌파봉 거래량 Z≥2(기존 1.5배 대체) · 윗꼬리 ≤30% · 손절을 "박스 중간값" → **박스 상단 -1% 재이탈**로 타이트하게 · EMA20 청산 유지
- rangeswing(하단 매수)과 방향이 반대인 별개 전략으로 공존 — 동일 종목 중복 진입은 공통 필터(§5-1d)가 차단

**S8. rebound — (하락) 과매도 반등** `1d · BEAR · risk4 · maxHold 2`
- 진입 필수: 5일 -8%↑ 또는 10일 -12%↑ 하락 · RSI(14)≤25 · 볼린저 하단 이탈 후 **밴드 안 재진입 봉** · 거래량 Z≥2.5(투매 확인) · 반등캔들(긴 아래꼬리/장악형) · 갭하락 -5% 초과 제외
- 손절: 반등봉 저가 이탈 또는 -2.5%
- 익절: +2.5% 40% → +5% 30% → 잔여 MA5 이탈 전량
- 시간청산: **2거래일 경과 무조건 전량**(수익 무관)
- positionPct 기존보다 하향(예 10%) — 위험도 반영
- 고도화 요지: 기존 RSI(2) 단일 → 투매(거래량)+재진입 확인형. "추세전환 기대 금지" 원칙을 2일 강제 청산으로 기계화

**S9. disparity — (하락) 낙폭과대 반등 [이격도 통합]** `1d · BEAR · risk3 · maxHold 5`
- 진입 필수: 5일 -15%↑ 또는 10일 -20%↑ 하락 · (종가 ≤ MA60×0.80 **또는** 종가/EMA25 ≤ 0.85 — 기존 이격도 조건 흡수) · RSI(14) 20~35(10 이하 제외: 추세 지속 위험) · 볼린저 하단 이탈→재진입 · CCI -200 이하 찍고 -100 상향 회복 · Slow Stoch 20 이하 골든크로스
- 손절: -3% 또는 반등봉 저가
- 익절: +5% 30% → +8% 30% → 잔여 +12% 또는 MA20 접근 시 전량
- 시간청산: 5거래일 전량
- 고도화 요지: 기존 disparity(이격도 낙주)와 낙폭과대 md의 조건을 합집합이 아닌 **정선된 교집합+택일 구조**로 재설계, 과매도 반등(S8)과는 낙폭 깊이·보유기간으로 명확히 층 분리(-8%/2일 vs -15%/5일)

## 6. Phase 4 — 관리자 전략관리 개편 (요구사항 6)

1. DB: `alter table bnf_strategies add column sort_order int not null default 100, add column regime text not null default 'ANY';` + 9전략 초기값 update.
2. AdminPage: HTML5 DnD(외부 라이브러리 금지)로 순서 변경 → sort_order 일괄 저장. 각 행에 장세 뱃지(📈/↔️/📉) + risk 표시 — 9전략 전부.
3. `STRATEGY_ORDER` 하드코딩 → `bnf_strategies.sort_order` 정렬로 대체(AuthContext 조회 확장). 게스트는 하드코딩 폴백.
- AC: 관리자 순서 변경 → 대시보드/스캐너 드롭다운 반영. 추천 로직도 이 순서를 우선순위로 사용.
