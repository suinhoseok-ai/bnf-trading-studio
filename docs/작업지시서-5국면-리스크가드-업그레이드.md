## 2. Phase A — 시장국면 엔진 v2 (설계서 01 기반)

### A-1. 지표 (일봉, range '2y' — SMA200+기울기 계산에 최소 260봉 필요)

```
SMA50, SMA200, SLOPE200 = SMA200/SMA200[20] - 1
ROC60, ROC120, ADX14(+DI/-DI, Wilder — 기존 adxArr 재사용), ATR_PCT14
BREADTH50 = KOSPI 유니버스 중 close>SMA50 비율 (서버 전용, 없으면 null)
```

### A-2. 후보 국면 판정 (우선순위 순서대로 평가, 첫 충족을 채택)

| 순서 | 국면 | 조건 |
|---|---|---|
| 1 | `BULL_MAJOR` | close>SMA200 · SMA50>SMA200 · SLOPE200≥+0.5% · ROC120≥+15% · ADX≥25 · +DI>-DI · (breadth null 또는 ≥60%) |
| 2 | `BEAR_MAJOR` | 위의 대칭 (close<SMA200 · SLOPE200≤-0.5% · ROC120≤-15% · -DI>+DI · breadth≤40%) |
| 3 | `BULL` | close>SMA200 · SMA50≥SMA200 · SLOPE200>0 · ROC60>0 · trend_up_score≥5 |
| 4 | `BEAR` | 대칭 (trend_down_score≥5) |
| 5 | `RANGE` | \|close-SMA200\|/SMA200≤3% · ADX<20 · \|ROC60\|≤5% |
| 6 | `TRANSITION` | 어느 것도 미충족 또는 신호 충돌 |

trend_up/down_score: 설계서 01 §5의 2+2+1+1+1+1 가중 투표(총 8점, breadth 없으면 7점 만점으로 동적 조정).
`confidence` = 채택 국면의 충족 가중치 합 / 가능한 가중치 합 (0~1).

### A-3. 확정(안정화) 규칙 — 기존 히스테리시스 대체

```
- 장전 세션에서만 확정 판단 수행 (하루 1회)
- candidate == confirmed → 유지, streak=0
- candidate가 전일 candidate와 같으면 streak+1, 다르면 streak=1
- streak >= N(기본 3, 하향 전환은 2) → 새 국면 확정
- 그 전까지 confirmed = TRANSITION (단, 직전 확정이 유지 조건을 아직 충족하면 유지)
- BULL_MAJOR ↔ BEAR_MAJOR 직접 전환 금지 (최소 1일 TRANSITION 경유)
```

### A-4. 비상 리스크오프 (`risk_state`)

```
- KOSPI 당일 등락률 ≤ -7% (설정값) → risk_state='EMERGENCY_RISK_OFF' 즉시 기록
  (확정 대기 없음 · 장중 세션에서도 감지 → trader가 즉시 신규 매수 전면 차단)
- 데이터 이상(최신 일봉 1거래일 이상 지연, 종가 결측) → 'DATA_INVALID' → 신규 매수 차단
- 정상 → 'NORMAL'
```

### A-5. DB 마이그레이션 (`bnf_market_regime`)

```sql
alter table public.bnf_market_regime
  drop constraint if exists bnf_market_regime_kospi_regime_check,
  drop constraint if exists bnf_market_regime_kosdaq_regime_check;
update public.bnf_market_regime set kospi_regime='RANGE' where kospi_regime='SIDEWAYS';
update public.bnf_market_regime set kosdaq_regime='RANGE' where kosdaq_regime='SIDEWAYS';
alter table public.bnf_market_regime
  add constraint bnf_market_regime_kospi_regime_check check (kospi_regime in ('BULL_MAJOR','BULL','RANGE','BEAR','BEAR_MAJOR','TRANSITION')),
  add constraint bnf_market_regime_kosdaq_regime_check check (kosdaq_regime in ('BULL_MAJOR','BULL','RANGE','BEAR','BEAR_MAJOR','TRANSITION')),
  add column if not exists kospi_candidate text,
  add column if not exists kosdaq_candidate text,
  add column if not exists confidence numeric,          -- KOSPI 기준
  add column if not exists confirmation_streak int not null default 0,
  add column if not exists risk_state text not null default 'NORMAL';
```

기존 코드의 `Regime` 타입(`marketRegime.ts`)과 이를 import하는 모든 곳(trader/regime.mts/Dashboard) 동시 갱신. `SIDEWAYS` 리터럴 전수 검색·치환.

### A-6. AC
- 과거 데이터로 5국면 각각 재현(수동 백테스트 스크립트 또는 콘솔 검증) · TRANSITION 경유 규칙 동작
- -7% 급락일 입력 시 EMERGENCY_RISK_OFF 기록 · 텔레그램 국면 알림에 신뢰도·전환 경과일 포함

## 3. Phase B — 전략 메타 5국면 확장 + 라우팅

### B-1. StrategyModule 확장 (`strategies/types.ts`)

```ts
export type Regime5 = 'BULL_MAJOR'|'BULL'|'RANGE'|'BEAR'|'BEAR_MAJOR'|'TRANSITION';
// 기존 regime(단일) → 표시용 primaryRegime으로 유지, 게이트·선정은 regimeFit 사용
regimeFit: Partial<Record<Regime5, number>>; // 0~100, 미기재=0(금지)
```

게이트 규칙: **fit ≥ 50이면 해당 국면에서 매수 허용**, 50 미만이면 차단.

### B-2. 9전략 × 5국면 적합도 매핑 (설계서 08 §7.1을 현행 전략에 이식·확정)

| 코드 | 설계서 대응 | BULL_MAJOR | BULL | RANGE | BEAR | BEAR_MAJOR |
|---|---|---:|---:|---:|---:|---:|
| breakout (추세추종·신고가돌파) | MAJOR_BULL_BREAKOUT+VCP 통합 | **100** | 85 | 15 | 0 | 0 |
| alignment (정배열) | — (자체) | 90 | **95** | 20 | 0 | 0 |
| pullback (눌림목 스윙) | BULL_PULLBACK | 80 | **100** | 25 | 0 | 0 |
| openbrk (시가돌파 단타) | — (자체, risk5) | 85 | 75 | 10 | 0 | 0 |
| bnf1 (볼린저 수렴회귀) | RANGE_BB_MEAN_REVERSION | 15 | 30 | **100** | 10 | 0 |
| rangeswing (박스권 스윙) | RANGE_SUPPORT_RESISTANCE 통합 | 10 | 25 | **95** | 10 | 0 |
| box (박스권 돌파) | — (횡보→상승 전환 포착) | 40 | 70 | **75** | 0 | 0 |
| rebound (과매도 반등) | BEAR_OVERSOLD_BOUNCE | 0 | 10 | 15 | **85** | 0 |
| disparity (낙폭과대 반등) | — (자체) | 0 | 10 | 15 | **80** | 0 |

- `TRANSITION`: 전 전략 0 (신규 매수 없음 — 기존 포지션 관리만).
- `BEAR_MAJOR`: 전 전략 0 (설계서 06 — 일반 롱 신규 진입 금지).
- 각 전략 모듈의 규칙 요약(rules)에 적합 국면 문구 반영.

### B-3. 국면별 익스포저 캡 (trader.mts 매수 루프 게이트, 설계서 07 §3)

```
BULL_MAJOR 100% · BULL 70% · RANGE 40% · BEAR 20% · BEAR_MAJOR 10% · TRANSITION 30%
검사식: (현재 자동매매 투입액 + 신규 주문액) / (투입액 + 예수금) <= 캡 → 초과 시 스킵+로그
```

### B-4. 하락 국면 라우팅 (DEFENSE 동작 — 설계서 05/06을 trader 규칙으로)

- `BEAR` 확정: fit<50 전략 전면 매수 차단(기존 게이트 확장), rebound/disparity만 소액 허용(캡 20% 내).
- `BEAR_MAJOR` 확정: **전 전략 신규 매수 차단**. 기존 포지션은 전략별 청산 로직 그대로 동작 + 텔레그램으로 "보유 포지션 점검 권고" 1회 발송. **자동 전량청산은 Phase D의 옵트인 설정을 따를 때만** 실행.
- `EMERGENCY_RISK_OFF` / `DATA_INVALID`: 국면 무관 신규 매수 전면 차단.

### B-5. AC
- 각 국면 목업으로 게이트 동작 확인(BULL에서 rebound 차단, RANGE에서 breakout 차단 등)
- 익스포저 캡 초과 시 매수 스킵 로그 · BEAR_MAJOR에서 어떤 전략도 매수 안 됨

## 4. Phase C — 전략선정엔진 라이트 (설계서 08 축약판)

전체 5요소 점수(신호품질·국면성과 등)는 실거래 데이터 축적 전엔 계산 불가하므로, **계산 가능한 3요소로 축약**하고 구조는 확장 가능하게 둔다.

### C-1. 미세상태 태그 (KOSPI 일봉에서 계산 — `marketRegime.ts`에 추가)

구현 대상 8종: `TREND_ACCELERATION` `VOLATILITY_COMPRESSION` `ORDERLY_PULLBACK` `RANGE_REVERSION` `OVERSOLD_PANIC` `TREND_BREAKDOWN` `OVERHEATED` `VOLATILITY_SHOCK` (판정식은 설계서 08 §6.2 그대로, BB폭 백분위·ATR% 중앙값 비교 포함). LEADER_BREADTH·SUPPORT_TEST·DATA_OR_LIQUIDITY_RISK는 제외(데이터 부족, DATA는 risk_state로 대체).

### C-2. 점수식 (0~100)

```
total = regimeFit(현 확정국면)                       -- 기본 골격
      + 미세상태 보정 (전략별 가중, 설계서 08 §10 표에서 해당 전략만 이식, 합산 ±30 한도)
      - 패널티:
          risk 4~5 전략             -15
          confidence < 0.55         -15 (공격형만)
          국면 전환 후 1일차          -10
          해당 전략 최근 3연속 손실    -15 (bnf_live_trades에서 계산)
          VOLATILITY_SHOCK          -25 (전 진입형)
```

### C-3. 1·2순위 선정 규칙

```
- BEAR_MAJOR → 1순위 '관망·현금화'(CASH_MODE 고정), 2순위 없음
- BEAR → 1순위 '리스크오프(현금 비중 확대)', 2순위 = rebound/disparity 중 고득점(60점 미만이면 CASH_MODE)
- TRANSITION → 1순위 '기존 포지션 관리', 2순위 CASH_MODE
- 그 외 → 총점 1위=1순위(60점 미만이면 CASH_MODE), 2순위는 55점 이상 & 1순위와 다른 성격
  (돌파형↔눌림형↔평균회귀형 구분: breakout/box/openbrk=돌파, pullback/alignment=눌림, bnf1/rangeswing=회귀)
- risk 5(openbrk)는 1·2순위 추천에서 항상 제외 (기존 방침 유지)
```

### C-4. 저장·표시

- `bnf_market_regime.detail`에 `selection: {primary, secondary, scores[], microTags[]}` JSON으로 포함(신규 테이블 없이 — 설계서 13장의 별도 테이블은 보류).
- 대시보드 시장현황 확장 패널: 1·2순위 + 각 점수 구성(적합도/보정/패널티) + 미세상태 태그 뱃지 + "왜 이 전략인가" 사유 문구(설계서 17.2 스타일).
- 텔레그램 국면 알림 본문에 1·2순위 반영 (기존 recommendLine 대체).

### C-5. AC
- 국면·태그 목업별 선정 결과가 §C-3 규칙과 일치 (BEAR_MAJOR→CASH, BULL+ORDERLY_PULLBACK→pullback 1순위 등)
- 점수 구성요소가 UI에서 재현 가능

## 5. Phase D — 리스크가드 + 장세 자동필터 (사진 스펙의 주식 변환)

### D-1. 설정 스키마 (`bnf_trading_settings`에 컬럼 추가)

```sql
alter table public.bnf_trading_settings
  add column if not exists rg_daily_loss_enabled boolean not null default false,
  add column if not exists rg_daily_loss_pct numeric not null default 3,      -- 총예산 기준 %
  add column if not exists rg_circuit_enabled boolean not null default true,
  add column if not exists rg_circuit_drop_pct numeric not null default 5,    -- KOSPI 당일 등락률 기준
  add column if not exists rg_circuit_block_hours int not null default 12,
  add column if not exists rg_circuit_until timestamptz,                       -- 발동 시 차단 만료 시각
  add column if not exists rg_streak_enabled boolean not null default false,
  add column if not exists rg_streak_losses int not null default 3,
  add column if not exists rg_streak_block_hours int not null default 24,
  add column if not exists rg_symbol_cooldown_enabled boolean not null default true,
  add column if not exists rg_symbol_cooldown_hours int not null default 24,
  add column if not exists rg_bear_major_liquidate boolean not null default false;

alter table public.bnf_trading_strategies
  add column if not exists regime_filter_enabled boolean not null default true;
```

### D-2. 4종 가드 동작 (사진 문구 그대로 이식, 코인→주식 변환. **모두 "신규 매수만" 차단, 매도/청산은 항상 동작**)

| # | 가드 | 주식 변환 스펙 | trader.mts 검사 위치 |
|---|---|---|---|
| ① | 일일 손실 한도 | 당일(KST) `bnf_live_trades` 매도 실현손익 합 ≤ -(전략 예산합계 × N%) → **당일** 신규 매수 중지 | 매수 루프 진입 전 1회 |
| ② | 서킷브레이커 (급락 감지) | BTC 4시간봉 → **KOSPI 당일 등락률**이 -N% 이하(기본 -5%) → `rg_circuit_until = now + N시간` 기록, 만료까지 전 전략 신규 매수 차단 | 매수 루프 진입 전 (발동 기록은 regime.mts/trader 중 먼저 감지한 쪽) |
| ③ | 연속 손절 쿨다운 | 같은 전략의 최근 매도 N회 연속 실현손실 → 그 전략만 N시간 매수 정지 | 전략 카드 루프 내 |
| ④ | 동일 종목 재진입 쿨다운 | 손절(SELL_SL) 매도된 종목은 N시간 동안 **어느 전략도** 재매수 금지 | 신호 수집 시 heldSymbols처럼 제외 |

- Phase A의 EMERGENCY_RISK_OFF(-7%)와 ②는 별개(②는 사용자 설정형·완화 기준, EMERGENCY는 시스템 고정 최후 방어선). 발동 시 각각 텔레그램 알림 + `bnf_trade_logs` 기록.

### D-3. 대세하락장 자동 전량청산 (기본 OFF — 사진의 경고 박스 그대로)

```
rg_bear_major_liquidate == true AND confirmed_regime이 BEAR_MAJOR로 새로 확정된 시점:
  → 자동매매가 보유 중인 bnf_live_positions(OPEN) 전체를 시장가 매도
  → 단, regime_filter_enabled=false인 전략 카드의 포지션은 제외 (사진 명세)
  → 수동 보유 종목(bnf_user_positions)은 건드리지 않음
  → 실행 전 텔레그램 통지, 실행 후 결과 통지
트리거 판정: trader.mts가 직전 회차의 confirmed와 비교해 "새로 확정" 전이를 감지 (중복 실행 방지 플래그 필요)
```

### D-4. 전략 카드별 "장세 자동필터" (TradingPage 카드에 체크박스 추가 — 사진 하단)

## 6. Phase E — UI 마무리

1. **대시보드 시장현황 카드 v2**: **기존 카드 UI 패턴을 그대로 유지** — 평소엔 다른 요약카드와 같은 크기로 접혀 있고(`regimeExpanded=false`), 클릭하면 하단에 풀폭 패널이 펼쳐지는 현행 구조(`DashboardPage.tsx`의 "시장현황" 카드 + 확장 패널)를 재사용한다. 새 레이아웃을 만들지 않는다.
   - **접힌 카드**: 5국면 아이콘(🚀 대세상승 / 📈 상승 / ↔️ 횡보 / 📉 하락 / 🧊 대세하락 / 🔄 전환구간) + KOSPI/KOSDAQ 요약(기존과 동일 형태), risk_state가 EMERGENCY/DATA_INVALID면 카드 테두리를 빨간색으로 강조.
   - **확장 패널**(기존 구조 그대로, 내용만 교체): 국면별 evidence 체크리스트(현행 유지) + 신뢰도%·전환 경과일(streak) 텍스트 추가 + **Phase C의 1순위·2순위 추천 전략 2개**(현행 "추천 전략" 한 줄 표시를 1·2순위 카드형으로 확장하되, 기존처럼 KOSPI/KOSDAQ 섹션 안에 자연스럽게 포함) + 최근 7일 국면 추이 뱃지(현행 유지, 아이콘만 5국면 체계로 교체) + 접기 버튼(현행 유지).
2. **관리자 전략관리**: 기존 국면 뱃지를 5국면 표기로 갱신 + **위험도 표시 추가**(risk 1~5를 "위험도 ●●●○○" 형태) — 사전확인 6번 항목.
3. **regime DB 값 표기**: 과거 SIDEWAYS 데이터가 RANGE로 마이그레이션되므로 7일 추이 뱃지도 새 아이콘 체계로 자동 호환.

## 8. 결정 필요 사항 (기본값으로 진행, 변경 원하면 지시)

1. **BEAR 확정 시 기존 보유 포지션 자동 축소**(설계서 05 A-3): 기본값 **하지 않음** — 신규 매수 차단+텔레그램 권고만. 자동 매도는 D-3 옵트인(BEAR_MAJOR 한정)으로만.
2. 인버스 ETF 헤지(설계서 06 전략 B): 기본값 **미구현** (설계서 스스로 1버전 비활성 권고).
3. 서킷브레이커 기준: 코인 -8%/4h → 주식 **KOSPI 당일 -5%**(설정 가능)로 완화 변환. EMERGENCY(-7%)는 별도 고정.
4. KOSDAQ 국면: 판정·표시는 유지하되 게이트·선정·캡은 **KOSPI 기준**(기존 방침 유지).
